> 文档版本：v1.0

---

## 1. 背景与目标

### 1.1 背景
当前培养基的用户流程是：输入主题 → AI 回答 → 生成子节点 → 追问延伸。

用户希望能够**上传已有的知识库文档**（PDF、Word、Markdown 等），让 AI 先解析文档结构，生成一个预制的思维导图，每个节点附带综述，帮助用户快速了解从哪里开始学习。

### 1.2 目标
- 用户上传知识库文档
- AI 自动解析文档，提取知识结构
- 生成预制思维导图，每个节点包含综述
- 用户点击任意节点即可开始深入学习和追问

---

## 2. 用户流程

```
[用户上传文档]
       ↓
[系统解析文档内容]
       ↓
[AI 分析结构，生成思维导图]
       ↓
[展示预制思维导图，每个节点有综述]
       ↓
[用户点击节点] → [开始学习/追问] → [延伸子节点]
```

## 3. 功能需求

### 3.1 文档上传

| 需求 | 说明 |
|------|------|
| 支持格式 | PDF、DOCX、Markdown、TXT |
| 文件大小限制 | 10MB |
| 上传方式 | 拖拽上传 + 点击选择文件 |
| 交互反馈 | 显示上传进度（解析中...） |

### 3.2 文档解析

| 需求 | 说明 |
|------|------|
| 文本提取 | 从 PDF/DOCX 中提取纯文本 |
| 长度处理 | 超过 8000 字截断，提示用户 |
| 格式保留 | 保留段落结构，供 AI 分析 |

### 3.3 AI 分析与思维导图生成

| 需求 | 说明 |
|------|------|
| 分析方式 | 调用 MiniMax API 分析文档结构 |
| 节点数量 | 4-8 个主要知识点 |
| 节点内容 | 每个节点包含：标题 + 综述（50-100字） |
| 综述要求 | 说明用户从这个节点能学到什么 |

**AI 提示词设计：**
```
你是一个知识结构分析专家。请分析提供的文档内容，生成一个思维导图结构。

## 任务
1. 分析文档的主要内容主题和结构
2. 提取 4-8 个主要知识点作为子节点
3. 为每个节点生成简短的综述（50-100字）

## 输出格式（JSON）
{
  "root": {
    "title": "文档主题",
    "summary": "文档的总体概述，100字左右"
  },
  "nodes": [
    {
      "title": "知识点标题1",
      "summary": "这个知识点的综述，50-100字，说明用户学习这个点能获得什么"
    }
  ]
}
```

### 3.4 思维导图展示

| 需求 | 说明 |
|------|------|
| 展示形式 | 与现有思维导图一致 |
| 节点内容 | 显示标题 + 综述（悬停/点击查看完整综述） |
| 来源标注 | 显示来源文件名 |

### 3.5 节点详情面板

| 需求 | 说明 |
|------|------|
| 综述展示 | 导入节点默认显示蓝色边框的综述卡片 |
| 来源标注 | 显示文档来源 |
| 追问入口 | 保留现有的追问输入框 |
| 答案区域 | 追问后在此展示回答 |

### 3.6 追问延伸

| 需求 | 说明 |
|------|------|
| 交互一致 | 与现有追问逻辑完全一致 |
| 上下文 | 传入节点标题作为上下文 |
| 流式输出 | 支持流式输出 + Markdown |

## 4. 数据模型

### 4.1 节点扩展字段

```typescript
interface KnowledgeNode {
  id: string;
  parentId: string | null;
  question: string;      // 主题/问题
  summary?: string;       // 综述（导入模式使用）
  answer: string;        // 回答
  concepts: string[];
  children: string[];
  createdAt: number;
  isStreamed: boolean;
  isExpanded?: boolean;
  isImported?: boolean;   // 是否为导入模式生成
  sourceFile?: string;   // 来源文件
}
```

### 4.2 思维导图结构

```typescript
interface KnowledgeTree {
  rootId: string | null;
  nodes: Record<string, KnowledgeNode>;
  currentNodeId: string | null;
  collapsedNodeIds: string[];
  isImported?: boolean;  // 标识是否为导入模式
}
```

## 5. API 设计

### 5.1 上传并解析文档

```
POST /api/knowledge-base/upload
Content-Type: multipart/form-data

参数：
  - file: 文件二进制

响应：
{
  "success": true,
  "conversationId": "xxx",
  "tree": { ... },
  "stats": {
    "totalNodes": 6,
    "fileName": "xxx.pdf",
    "contentLength": 5000
  }
}

错误响应：
{
  "error": "不支持的文件格式"
}
```

### 5.2 解析飞书文档（可选）

```
POST /api/knowledge-base/feishu
Content-Type: application/json

参数：
{
  "docToken": "飞书文档token"
}

响应：同上
```

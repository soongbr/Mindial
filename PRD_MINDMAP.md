# 培养基 - 思维导图式知识可视化 PRD

## 1. 背景与目标

### 现状问题
- 当前知识树是简单的垂直树状列表，层级关系不直观
- 左侧知识树在点击节点后会收起，无法同时查看多个层级
- 节点展开/收起的行为不符合用户预期（应该手动控制）

### 用户期望
参考 XMind/GitMind 等思维导图工具，实现：
- **水平思维导图布局**，从左到右展示知识层级
- **点击节点** → 打开详情页（对话+知识讲解）
- **展开/折叠按钮** → 手动控制子节点的显示/隐藏
- **颜色分层** → 不同层级用不同颜色区分

## 2. 产品设计

### 2.1 整体布局

```
┌─────────────────────────────────────────────────────────────────┐
│  Header: 培养基 - AI知识探索工具                    [联网搜索] │
├───────────────┬─────────────────────────────────────────────────┤
│               │                                                 │
│   思维导图区域  │           节点详情面板                          │
│   (左侧60%)   │           (右侧40%)                            │
│               │                                                 │
│   ┌───┐       ┌───┐       ┌───┐                                │
│   │根 │───>───│一级│───>───│二级│                              │
│   └───┘       └───┘       └───┘                                │
│     │           <展开>                                          │
│     │           ┌───┐                                          │
│     └─────>────│二级│                                          │

│  输入框: 输入主题开始探索                              [开始]  │
└─────────────────────────────────────────────────────────────────┘
```

### 2.2 思维导图节点样式

| 层级 | 背景色 | 文字色 | 圆角 | 说明 |
|------|--------|--------|------|------|
| 根节点 (Level 0) | `#E8D5F5` 淡紫色 | `#000000` | `16px` | 主题/问题 |
| 一级节点 (Level 1) | `#D5E8F5` 淡蓝色 | `#000000` | `12px` | 追问/子问题 |
| 二级节点 (Level 2) | `#D5F5E8` 淡绿色 | `#000000` | `12px` | 更深层的追问 |
| 三级+节点 | `#F5F5DC` 淡黄色 | `#000000` | `8px` | 深层追问 |

### 2.3 连接线样式

- **类型**：贝塞尔曲线（柔和的曲线）
- **颜色**：与父节点颜色一致（或淡灰色 `#CCCCCC`）
- **粗细**：`2px`
- **风格**：从父节点右侧出发，连接到子节点左侧

### 2.4 展开/折叠按钮

- **位置**：连接线的分叉点处（小圆点）
- **图标**：
  - `>` 表示有子节点但已折叠（可展开）
  - `<` 表示有子节点且已展开（可折叠）
  - 无图标表示叶子节点（无子节点）
- **尺寸**：圆形按钮，直径 `20px`
- **交互**：点击切换展开/折叠状态

## 3. 功能需求

### 3.1 思维导图渲染

**FR-1**: 知识树以水平思维导图形式展示，根节点在最左侧，子节点向右延伸

**FR-2**: 不同层级使用不同的背景色（见 2.2 表格）

**FR-3**: 节点文字过长时显示省略号，悬停显示完整文字

**FR-4**: 当前选中的节点有明显的边框/阴影高亮

### 3.2 展开/折叠控制

**FR-5**: 每个有子节点的节点旁边显示展开/折叠按钮

**FR-6**: 点击按钮可以手动展开或折叠子节点

**FR-7**: 展开/折叠状态需要**持久化保存**（存储到 JSON）

**FR-8**: 默认行为：
- 根节点**展开**
- 一级节点**折叠**
- 用户手动展开后保持展开状态

### 3.3 节点点击

**FR-9**: 点击节点 → 右侧面板显示该节点的详情

**FR-10**: 详情面板包含：
- 节点问题
- AI 回答（支持 Markdown 渲染）
- 关键概念标签
- 追问输入框

### 3.4 节点追问

**FR-11**: 在详情面板的追问输入框输入内容 → 创建新的子节点

**FR-12**: 新子节点自动添加到思维导图中，并展开父节点

### 3.5 知识树数据持久化

**FR-13**: 每次节点变化（创建/更新/删除）→ 保存到 `backend/data/knowledge-tree.json`

**FR-14**: 页面加载时从 JSON 恢复知识树（包括展开/折叠状态）

## 4. 技术方案

### 4.1 技术选型

| 模块 | 技术 | 说明 |
|------|------|------|
| 前端框架 | React + TypeScript | 现有项目延续 |
| 思维导图渲染 | React Flow 或 D3.js | 二选一，优先 React Flow（更易用） |
| 状态管理 | React useState + Context | 现有项目延续 |
| 后端 | Node.js + Express | 现有项目延续 |
| 存储 | JSON 文件 | 现有项目延续 |

### 4.2 备选方案对比

| 方案 | 优点 | 缺点 |
|------|------|------|
| **React Flow** | 开源、成熟、支持自定义节点、文档完善 | 需要学习 API |
| **D3.js** | 高度自定义、灵活 | 开发量大、代码复杂 |

**推荐**：优先使用 **React Flow**，可以快速实现思维导图功能。

### 4.3 数据模型更新

```typescript
interface KnowledgeNode {
  id: string;
  parentId: string | null;
  question: string;
  answer: string;
  concepts: string[];
  children: string[];      // 子节点 ID 数组
  createdAt: number;
  isStreamed: boolean;
  isExpanded?: boolean;    // 新增：展开/折叠状态

interface KnowledgeTree {
  rootId: string | null;
  nodes: Record<string, KnowledgeNode>;
  currentNodeId: string | null;
  // 新增：存储折叠的节点 ID 数组
  collapsedNodeIds: string[];
}
```

## 5. 交互细节

### 5.1 思维导图区域

- **缩放**：支持鼠标滚轮缩放
- **平移**：支持拖拽画布平移
- **自动布局**：节点自动水平排列，保持适当间距

### 5.2 节点悬停

- 悬停时显示完整问题文字（tooltip）
- 悬停时节点略微放大（scale 1.02）

### 5.3 节点点击

- 单击 → 选中节点，右侧显示详情
- 选中的节点有 `2px solid #8B5CF6` 边框

### 5.4 展开/折叠按钮

- 悬停 → 背景色变深
- 点击 → 切换展开/折叠状态，同时触发动画

## 三、缺失功能清单

### 3.1 知识库侧边栏（必须新增）

**现状：** 没有统一的文档管理面板

**需求：**
- 左侧面板显示「我的知识库」
- 列出所有已上传文档（文件名、大小、上传时间）
- 每个文档操作：[生成导图] [删除]
- 顶部：「上传文档」按钮

**API 需求：**
```
GET /api/knowledge/documents
响应：{ documents: [{ id, fileName, fileSize, uploadedAt, hasMindMap }] }

DELETE /api/knowledge/documents/:id
响应：{ success: true }
```

---

### 3.2 模式切换机制（必须新增）

**现状：** 没有模式概念，上传就直接生成导图

**需求：**
- 右上角显示当前模式：「🔍 自由探索」或「📚 目标学习」
- 点击切换模式
- 切换时保留知识库状态

**UI 示例：**
```
┌─ 知识库 ───────────────────────┐
│ 📚 我的知识库 [🔍自由 ▼]       │  ← 模式切换下拉
│                               │
│ ✅ xxx.pdf [生成导图] [删除]   │
│ ✅ yyy.docx [生成导图] [删除]  │
│   zzz.md [生成导图] [删除]     │
│                               │
│ [📤 上传文档]                 │
└──────────────────────────────────┘
```

---

### 3.3 🔍 自由探索模式（新增）

**现状：** 完全未实现

**功能：**
1. 勾选文档（复选框）→ 作为参考资料
2. 用户自由输入问题
3. AI 回答时引用勾选的文档（RAG 检索）
4. 不生成导图，只返回答案
5. 侧边栏显示「已引用：xxx.pdf, yyy.docx」

**API 需求：**
```
POST /api/knowledge/search
Body: { query, documentIds?, topK, minSimilarity }
响应：{ results: [{ content, score, fileName }] }

GET /api/knowledge/reference-status
响应：{ referencedIds: ['doc_xxx', 'doc_yyy'] }
```

---

### 3.4 📚 目标学习模式（补充）

**现状：** 上传时自动生成导图，没有「生成导图」按钮

**需求：**
1. 文档列表默认不生成导图
2. 点击「生成导图」按钮 → 才开始 AI 解析
3. 加载状态：显示「正在分析文档结构...」
4. 生成后展示导图

**交互：**
```
✅ xxx.pdf [生成导图] [删除]
              ↑ 点击后
              ↓
         「正在分析文档结构...」（加载中）
              ↓
         展示导图 + 更新按钮状态
```

---

### 3.5 导图层级限制（补充）

**现状：** 没有限制，AI 生成多少算多少

**需求：**
- 1-3 级树状结构
- 每级节点数 ≤ 5 个
- 在 `documentParser.analyzeAndGenerateMindMap` 中处理

---

### 3.6 文档大小限制（调整）

**现状：** 前端 10MB，后端 50MB，不一致

**需求：**
- 统一限制：≤ 100MB
- 前端提示更新
